# Release v0.8.0 — Gestión de Pedidos para Administrador

## Resumen
La versión **v0.8.0** incorpora el módulo base de **gestión de pedidos para administrador** en el backend PHP del proyecto **Marucha**.  
En esta versión se implementaron endpoints para que el administrador pueda consultar todos los pedidos registrados, visualizar el detalle completo de un pedido y actualizar su estado, registrando además el historial correspondiente.

---

## Rama de desarrollo
- `feature/admin-orders`

---

## Objetivo de la versión
Extender el backend de Marucha para que el administrador pueda:

- consultar todos los pedidos registrados en el sistema
- visualizar el detalle completo de un pedido
- actualizar el estado de un pedido
- registrar el historial de cambios de estado del pedido

---

## Funcionalidades implementadas

## 1. Ampliación del modelo de pedidos
Se amplió `models/Order.php` para soportar las operaciones administrativas del módulo de pedidos.

### Responsabilidades agregadas
- obtener todos los pedidos del sistema
- obtener detalle administrativo de un pedido
- obtener estado de pedido por ID
- actualizar estado de un pedido

---

## 2. Servicio administrativo de pedidos
Se creó `services/AdminOrderService.php` para centralizar la lógica de negocio del módulo administrativo de pedidos.

### Responsabilidades implementadas
- obtener todos los pedidos del sistema
- obtener el detalle completo de un pedido para administrador
- validar existencia del pedido antes de cambiar estado
- validar existencia del estado de pedido
- actualizar el estado de un pedido
- registrar historial de cambio en `pedido_historial_estado`
- devolver el pedido actualizado con su dirección y detalle

---

## 3. Controlador administrativo de pedidos
Se creó `controllers/AdminOrderController.php` para exponer endpoints administrativos de pedidos.

### Endpoints implementados
```http
GET /api/admin/orders
GET /api/admin/orders/{id}
PUT /api/admin/orders/{id}/status
