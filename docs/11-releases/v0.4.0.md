# Release v0.4.0 — Authentication Base

## Resumen
La versión **v0.4.0** incorpora la base de autenticación del backend PHP del proyecto **Marucha**.  
En esta versión se implementó el flujo inicial de **registro** e **inicio de sesión** para usuarios locales, utilizando validaciones básicas, almacenamiento seguro de contraseñas mediante hash y respuestas JSON consumibles por la aplicación móvil.

---

## Rama de desarrollo
- `feature/auth-base`

---

## Objetivo de la versión
Implementar la primera capa de autenticación del backend, incluyendo:

- registro de usuarios clientes
- inicio de sesión con email y contraseña
- validaciones básicas de entrada
- verificación de duplicados por email y username
- almacenamiento de contraseñas con hash seguro
- endpoints JSON de autenticación

---

## Funcionalidades implementadas

### Validación
- `helpers/Validator.php`
  - validación de campos obligatorios
  - validación de formato de email
  - validación de longitud mínima de contraseña

### Modelo de usuario
- `models/User.php`
  - búsqueda de usuario por email
  - búsqueda de usuario por username
  - creación de nuevos usuarios en la tabla `usuario`

### Servicio de autenticación
- `services/AuthService.php`
  - lógica de registro
  - lógica de login
  - validación de credenciales
  - generación de hash con `password_hash()`
  - verificación de contraseña con `password_verify()`

### Controlador
- `controllers/AuthController.php`
  - manejo de requests de registro y login
  - lectura del body JSON
  - retorno de respuestas JSON estandarizadas

### Rutas agregadas
En `routes/api.php` se añadieron los endpoints:

```http
POST /api/auth/register
POST /api/auth/login
