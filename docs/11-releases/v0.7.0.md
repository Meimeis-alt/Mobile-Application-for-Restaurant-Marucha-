# Release v0.7.0 — Módulo Base de Pedidos

## Resumen
La versión **v0.7.0** incorpora el primer módulo completo de **pedidos** para el backend PHP del proyecto **Marucha**.  
En esta versión se implementó la lógica necesaria para convertir el carrito activo de un usuario en un pedido real, registrar el detalle del pedido, asignar método de pago y dirección de entrega, registrar el estado inicial del pedido y consultar el historial y detalle de pedidos.

---

## Rama de desarrollo
- `feature/orders-base`

---

## Objetivo de la versión
Extender el backend de Marucha para que un usuario pueda:

- convertir su carrito activo en un pedido
- registrar el detalle del pedido a partir de los productos del carrito
- asociar una dirección de entrega
- asociar un método de pago
- asignar un estado inicial al pedido
- registrar el historial de estado del pedido
- consultar su historial de pedidos
- consultar el detalle completo de un pedido

---

## Funcionalidades implementadas

## 1. Modelo de pedidos
Se creó `models/Order.php` para centralizar las operaciones relacionadas con el flujo de pedidos.

### Responsabilidades implementadas
- obtener métodos de pago
- obtener estados de pedido
- generar número de pedido
- crear pedido
- crear detalle de pedido
- registrar historial de estado
- obtener pedido por ID
- obtener pedidos por usuario
- obtener detalle de pedido
- obtener dirección asociada a un pedido

---

## 2. Integración con el modelo de carrito
Se amplió `models/Cart.php` para soportar el flujo de creación de pedidos.

### Funciones agregadas
- actualizar estado del carrito
- calcular total del carrito

Esto permite marcar el carrito como `convertido` una vez creado el pedido y reutilizar el total del carrito como subtotal/total del pedido.

---

## 3. Servicio de pedidos
Se creó `services/OrderService.php` para contener la lógica de negocio del módulo de pedidos.

### Responsabilidades implementadas
- validar existencia del usuario
- validar que la dirección exista y pertenezca al usuario
- validar existencia y estado del método de pago
- validar existencia del carrito activo
- validar que el carrito tenga productos antes del checkout
- crear cabecera del pedido
- copiar los productos del carrito a `detalle_pedido`
- registrar el estado inicial en `pedido_historial_estado`
- convertir el estado del carrito después del checkout
- obtener historial de pedidos por usuario
- obtener detalle completo de un pedido por ID

---
